## 复制粘贴package.json后的「正确操作步骤」（非常重要）

### ✅ 正确流程（照着做）

#### ① 覆盖 package.json

直接把 **新内容完整替换旧内容** ，package.json ≠ JS 文件，它是纯 JSON，只能剩一个完整 JSON 对象，链注释都不能有

#### ② 删除旧依赖node_modules（必须）

```
rm -rf node_modules package-lock.json
# 或 Windows:
# 删除 node_modules 文件夹 和 package-lock.json
```

#### ③ 重新安装

```
npm install
```

#### ④ 启动

```
npm run dev
```

> npm 报错里这句是关键：
>
> > ```
> > Note: package.json must be actual JSON, not just JavaScript.
> > ```
>
> 意思是：
>
> - ❌ 它不是 JS
> - ❌ 不认识 `//`
> - ❌ 不认识注释
> - ❌ 不认识多余内容
>
> **只要有一个非法字符，npm 就直接停**

**只要没动 `package.json`，就不用再 install**

可以直接：

```
npm run dev
```

## 复制粘贴vite.config.js后的「正确操作步骤」（非常重要）

### ① `path` 在某些环境下的问题（小概率）

你现在用的是：

```
import path from 'path'
```

在 **Vite 4 + Node ≥16** 基本没问题
 如果你哪天遇到报错（找不到 `__dirname`），可以换成 **ESM 标准写法**：

```
import { fileURLToPath, URL } from 'node:url'

resolve: {
  alias: {
    '@': fileURLToPath(new URL('./src', import.meta.url))
  }
}
```

### ② 代理只在 dev 生效（不是 bug）

# 提醒：

- `server.proxy` **只在 `npm run dev` 生效**
- `npm run build` 后不会自动代理

这是 Vite 的设计，不是你配错

**在 `.vue` 文件里：**

- ❌ 不要用 `//` 注释 `<template>` / `<script>` / `<style>`
- ✅ 不需要的代码就删
- ✅ 真的要留，用 Git，不用注释
